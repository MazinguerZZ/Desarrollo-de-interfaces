# Práctica 1.3 — Proyecto integrador HTML + CSS

**Módulo:** 2º DAM — Diseño de Interfaces · UD1 Planificación de interfaces

---

## Grupos y entrega

**Grupos de 2 personas.** Si un alumno queda solo puede unirse a un grupo existente. Máximo 3 personas.

### Repositorio GitHub

Cada alumno sube el trabajo a su propio repositorio con esta estructura:
```
Proyecto_Interfaz/
├── index.html
├── página2.html
├── página3.html
├── css/
│   └── (archivos de estilos)
└── README.md
```

El `README.md` debe incluir una breve descripción del proyecto y los pasos para ejecutarlo.

### Archivo PDF

Cada alumno entrega en el aula virtual `practica1.3_nombreapellido.pdf` con:

1. **Portada** — captura de la versión final de la interfaz y miembros del grupo.
2. **Enlace a GitHub.**
3. **Pasos y respuestas** de la Parte 1 y la Parte 3.

---

## Objetivo

Integrar los conocimientos adquiridos durante el RA1 del módulo:

- Planificar y diseñar una interfaz multiplataforma con wireframes y prototipos funcionales.
- Aplicar principios de maquetación con HTML5 y CSS3.
- Implementar layouts con Flexbox y Grid.
- Integrar transiciones, transformaciones y posicionamientos.
- Documentar el proceso en GitHub y presentar el resultado final.
- Reflexionar sobre la evolución de la interfaz hacia un modelo NUI.

---

## Parte 1 — Diseño y prototipado

**Herramientas:** Balsamiq, Proto.io o Wireframe.cc

- Diseña una interfaz estática (web, app o panel) con estructura clara: cabecera, cuerpo y pie.
- Crea un **wireframe** en Balsamiq (estructura básica) y un **prototipo clicable** en Proto.io.
- Justifica tus decisiones de diseño: colores, tipografía y jerarquía visual.
- Entrega una breve presentación de **2–3 diapositivas** explicando el diseño y el público objetivo.

---

## Parte 2 — Desarrollo en HTML + CSS

Convierte tu prototipo en una interfaz real. El código debe incluir:

| Requisito | Detalle |
|-----------|---------|
| Archivos HTML | Al menos **3 páginas** `.html` |
| Comentarios | En los principales bloques de código |
| Semántica | Etiquetas HTML5 (`header`, `main`, `footer`, `section`…) |
| Layout | Flexbox y Grid, incluyendo `grid-template-areas` o `justify-content` |
| Posicionamiento | `relative`, `absolute`, `fixed` y `sticky` |
| Animaciones | `transition` y `transform` |
| Responsive | `@media queries` |

---

## Parte 3 — Exploración NUI *(Natural User Interface)*

Reflexiona sobre cómo sería tu interfaz si se transformara en una NUI. Incluye una **diapositiva breve** respondiendo a:

- ¿Qué partes podrían controlarse mediante **voz, gestos o sensores**?
- ¿Qué cambios de diseño requeriría para una versión **móvil táctil o basada en movimiento**?
- ¿Cómo debería responder la interfaz **visual o sonoramente** al usuario?
