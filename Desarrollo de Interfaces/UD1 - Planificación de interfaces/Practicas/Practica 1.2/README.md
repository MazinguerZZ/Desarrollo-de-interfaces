# Práctica 1.2 — Revisión de ejemplos y fases de diseño de interfaz

**Módulo:** 2º DAM — Diseño de Interfaces · UD1 Planificación de interfaces  

---

## Parte 1 — Modificación de ejemplos HTML + CSS

**Entrega:** enlaces de GitHub con los documentos HTML modificados.

### Instrucciones

- Modifica los ejemplos vistos en clase de HTML + CSS sobre **tipos de selectores**.
- Cada modificación debe estar **comentada** en el propio documento, dentro del elemento `<style>`:
```css
/* Comentario explicando la modificación — Nombre Apellido */
```

- Puedes añadir más elementos HTML y propiedades CSS para que las modificaciones sean más visibles en el resultado.

## Parte 2 — Fases de diseño de interfaz

**Entrega:** los cuatro artefactos siguientes:

| Fase | Herramienta | Formato de entrega |
|------|-------------|--------------------|
| **Sketch** | Papel y lápiz | Foto o escaneado |
| **Wireframe** | [wireframe.cc](https://wireframe.cc) | Captura de pantalla |
| **Mockup** | [Balsamiq](https://balsamiq.com) | Captura o enlace |
| **Prototipo** | [Proto.io](https://proto.io) | Enlace |

### Instrucciones

Idea y diseña una **interfaz sencilla** de tu elección. Ejemplo: formulario de registro en una plataforma de videojuegos con campos de nombre, email, contraseña y botón *Enviar*.

- En esta práctica **no es necesario HTML ni CSS** — las herramientas son meramente visuales.
- Proto.io debe simular la **navegación entre pantallas**.
