## Instrucciones

Elabora un **diagrama del flujo de datos** de tu aplicación de la práctica 3.2.

### El diagrama debe incluir

Los siguientes componentes:

- `App` *(componente padre)*
- `Selector`
- `ListaEquipo`
- `Resumen`

### Flechas

- **↓ Hacia abajo** — desde `App` hacia los componentes hijos, indicando qué información reciben (**Props**).
- **↑ Hacia arriba** — desde los componentes hijos hacia `App`, indicando qué acciones solicitan al padre (**Funciones / Eventos**).

> Puedes entregar una foto de un esquema hecho a mano o un esquema digital (por ejemplo con LucidChart).

## Preguntas a responder

Debajo del esquema, responde las siguientes cuestiones:

1. ¿Dónde vive el **estado principal** de la aplicación?
2. ¿Por qué los componentes hijos **no modifican directamente** el estado?
3. ¿Qué significa que el flujo de datos sea **unidireccional**?
